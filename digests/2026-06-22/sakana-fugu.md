# Sakana Fugu

本文展示了一个 AI 智能体自主改进小型 GPT 训练方案的实验。该实验使用 AutoResearch（Karpathy 等人提出）——一种迭代编辑训练代码、运行实验并仅保留降低验证 bits-per-byte（BPB）的更改的智能体框架——在单块 H100 GPU 上用约 14 小时运行了 123 次实验。每条线描绘了随着实验累积，各系统所达到的最佳 BPB 的变化轨迹：Fugu-Ultra 以粗红线表示（实线 = 三个种子的平均值，虚线 = 最佳单次运行），其后以淡色叠加了三个前沿模型基线（Model A、B 和 C），标注则标记了智能体自行发现的每一项改进——涵盖批量大小、模型深度、学习率和优化器设置等方面。Fugu-Ultra 最终以最佳平均 BPB（0.9774 ± 0.0019）收官，超过了 Model C（0.9781）、Model B（0.9793）和 Model A（0.9822），其最佳单次运行达到 0.9748，领先所有基线。这表明编排多个强大模型可以在智能体式机器学习研究中超越任何单一前沿模型。

示例 2 — 假名消息的阅读顺序推定
本案例研究以假名消息（古典日语假名书信）这一历史文献的阅读顺序推定问题为对象。假名消息采用被称为"散らし書き"的形式，将文字散布在纸面上书写，因此即使是习惯阅读古文书的人也很难正确判断文字的阅读顺序。为此，向每个模型提供文字的边界框（bounding box）和大致的阅读顺序规则，让其输出推定文字阅读顺序的代码。实验选取了 1610 年芳春院（1547–1617）所写的书信作为对象，以 NED（基于与专家标注的正确阅读顺序的归一化编辑距离的分数，1.0 为完全匹配）进行评价。将多个前沿模型（A-C）通过同一流程处理后，Fugu-Ultra 的结果远远领先于其他模型。Model A 的 NED 为 0.24，Model B 也与之相差无几，均远不及 Fugu-Ultra 的 0.80。此外，Model C 未能输出任何有效代码。为了可视化模型间阅读顺序的差异，在专家标注的正确阅读顺序（绿色）上叠加了推定路径（红色）制作了视频。Fugu-Ultra（上方）几乎准确地追踪了阅读顺序，而 Model A（下方）则在整个纸面上四处跳跃，两者呈现出截然不同的结果。图：芳春院消息（庆应义塾大学斯道文库藏）

在本基准测试中，Fugu-Ultra 和 3 个前沿模型各自收到一个提示，要求从零开始用纯 Python 编写一个魔方求解器——不允许使用现成的求解库——生成的程序在本地对 300 个随机打乱的魔方组成的保留测试集上运行。解法质量以使用的步数衡量，步数越少越好。Fugu-Ultra 和前沿 Model A 编写的求解器成功运行并解决了所有 300 个魔方，而 Model B 和 Model C 各自生成了看似精巧的代码，却在执行时崩溃，未能返回任何有效解法（0/300）。视频跟踪了第 17 号魔方：从同一打乱状态出发，Fugu-Ultra 的求解器用 19 步达到了复原状态，而 Model A 需要 21 步——在全部 300 个魔方中，Fugu-Ultra 平均需要 19.72 步，Model A 为 19.76 步，两者均接近最优前沿，而 Fugu-Ultra 在任何一个魔方上的步数都不多于 Model A（7 胜、293 平、0 负）。
