原则
你下面将读到的一切，都是为了遵循以下三条原则：
- 不凭空创造数据。资金无法从无到有产生，因此我们不能容忍重复或任意的余额更新。我们通过幂等性、去重和对账来强制执行这一点。
- 不丢失数据。发生在资金上的一切都必须被追踪和持久化。我们通过完整精度、至少一次投递、事件溯源、审计轨迹和不可变性来保护这一点。
- 不信任。既不信任外部提供方，也不信任内部组件，更不信任外部世界。我们通过验证 webhook、跨来源交叉核对数据，以及在假设被打破时大声失败来坚守这一点。
表示货币
在你能够转移或记录资金之前，你必须先表示它。这些是关于如何对一个货币值进行建模、存储、计算和转换的决策。一旦弄错，其上的每一层都会继承这个错误。
精度处理
货币表示是金融系统中最基础的决策之一。主要有四种做法：
- 浮点数。内置的 float 或 double 类型。这会造成不可预测的精度损失，几乎从来都不是一个好主意。但它最快、内存效率最高，且不需要额外的库或数据结构。
- 任意精度。诸如 Java 的 `BigDecimal` 这样的类型让你能够精确地控制计算的精度。代码是可预测的，我们可以决定在何处以及如何进行舍入。它适合像外汇或定价计算这样的中间运算，因为其中许多运算会链式连接在一起。
- 最小单位精度。对于大多数法定货币，只保留固定精度是可以的，即与所连接的中央银行系统使用的精度相同。位数由 ISO 4217 描述（不要假设它总是 2 位，并非如此！）。在实践中，这意味着把金额存储为以其最小单位计的整数——€12.34 变成 `1234`。加密货币使用同样的“整数最小单位”思路（BTC 用 satoshi，ETH 用 wei），但有两点不同：精度是按资产而定的，由代币本身定义（例如 ERC-20 的 `decimals`），通常是 18 位；并且由此产生的数量级会溢出 64 位整数，因此你需要任意宽度的整数来容纳它们。
- 有理数。当不可接受任何精度损失时。这是最强大的方法，但也有其自身的弊端。第一，它比其他方法慢。第二，它在不损失精度的情况下无法转换为其他格式。第三，它通常需要自定义数据类型或库。
选择哪一种取决于系统的类别及其职责。这里没有什么经验法则，除了不要使用浮点数。这些表示方法之间也并不互斥——你如何存储一个金额与你如何对其进行计算是两个独立的决策，一个系统常常将它们结合起来，例如用整数存储，配合 `BigDecimal` 进行中间计算。
当一个金额被序列化时，同样需要这份谨慎。在大多数解析器中，一个裸的 JSON 数字就是一个 IEEE-754 double，因此把货币序列化为数字会在边界处重新引入浮点数问题，无论你在内部多么小心地表示它。要么把货币作为字符串（`"12.34"`）发送，要么作为以其最小单位计的整数发送。
触及的原则：
- 不丢失数据。错误的表示会悄无声息地丢弃永远无法恢复的精度。
舍入策略
- 舍入是不可避免的。它应当被显式地完成：任何除法、货币转换、费用、利息或费率应用，或在精度之间的转换，都可能需要舍入。
- 它是一个业务决策。不同的舍入策略有不同的影响。有时你必须保守（例如不要花费你没有的钱）而向下舍入；有时你关心统计效应而使用四舍六入五成双（half-even）。决定谁得到那个零头可能带来法律/税务上的影响。
- 尽可能少地舍入。你保持完整精度的时间越长，你在恰当的情境中做出正确决策的选择就越多。舍入通常应当发生在边界上，例如在数字被持久化或展示给用户之前。
- 舍入会破坏求和。如果一个数字被拆分成若干部分并施加舍入，各部分之和可能不再等于原始数字。视情境而定，这可能需要显式处理——例如设立一个专门的舍入账户。
触及的原则：
- 不丢失数据。残值必须被追踪，而不是被丢弃。
- 不凭空创造数据。舍入绝不能铸造出本不存在的钱。
货币处理
钱不能仅用一个数字来表示——它与一种货币成对出现。在处理货币时有一些细微之处。
- 把金额和货币打包在一起。一个 `Money` 新类型（struct、class、record 等）能最大限度地减少出错的机会。
- 不做跨币种算术。你的系统应当禁止把两个不同货币的金额相加。转换应当通过一个严格受控的汇率非常显式地进行。
- 使用一个受控的货币集合。可以是一个自定义配置项、JDK 数据库，或专门的服务。绝不接受任意的货币代码；在系统边界处进行验证。
- 代码仅标识法币。货币代码只对法币而言是唯一且可用作标识符的。对于加密货币，你将不得不使用更复杂的方式，例如 `(network, contract address)` 或类似形式。
- 货币携带元数据。符号、精度、名称等。你通常出于显示目的需要这些细节，但在业务逻辑中很少需要。
- 锚定的不等于其标的。锚定（pegged）、桥接（bridged）和封装（wrapped）的加密货币并不等同于其标的货币。
触及的原则：
- 不信任。在边界处对照受控集合验证货币。
- 不凭空创造数据。把不同的货币/资产当作可互换的，会凭空变出价值。
外汇汇率
外汇（Forex，外汇货币市场）汇率让我们能够在不同货币之间转换资金。
- 汇率总是有方向的。EUR/USD 汇率与其倒数 USD/USD 汇率并不是一回事。在交易所里，买入和卖出是两个不同价格的不同订单（买卖价差，bid/ask spread），因此这两个方向并不能简单地互为倒数。
- 汇率的时间至关重要。虽然技术上你可以使用任何时间点的汇率，但最常用的是：
- 当前时间汇率。用于计算当前持仓，或一笔交易仿佛此刻发生时的价值。
- 起息日汇率。用于计算价值变动或税额。
- 有两种汇率对转换有意义：
- 交易汇率。一次真实转换所发生的汇率。你不会直接存储它——它由原始金额和结果金额推导得出。
- 参考汇率（中间市场或中央银行）。用于估值和等价（持仓此刻值多少，或起息日的计税基础），而不是任何人实际交易的价格。
- 不存在权威汇率。汇率来自市场，并在不同交易场所或计算方法之间有所差异。最接近权威的是中央银行汇率，但它只能用作参考汇率，并且即便如此我们也可能有同样有效的替代来源。
