# Framework's 10G Ethernet module exposes USB-C's complexity

Framework 的 10G 以太网模块暴露了 USB-C 的复杂性

过去几年，我一直在关注 WisdPi 开发的各种 5 Gbps 和 10 Gbps 以太网适配器。

它们采用较新的 Realtek 以太网芯片，这些芯片有时会有性能上的怪异表现——最常见的是在 Linux 下遇到。

在今天的视频中，我测试了适用于 Framework 电脑的全新 WisdPi 10G 以太网扩展卡。它可以装入任何可用的 Framework 扩展槽——甚至包括 Framework Desktop。

但扩展卡使用 USB-C 与主板连接——问题就出在这里……

主要问题在于 USB-C 带宽的复杂性——尤其是当它与 Realtek RTL8159 以太网控制器搭配使用时，该控制器需要 USB 3.2 Gen 2x2（20 Gbps）才能达到完整的标称 10 Gbps 速度。

在许多 Framework 笔记本上，你最终得到的速度会明显低于 10 Gbps（实际最高 9.4 Gbps）：

上图显示的是我在一台搭载 AMD Ryzen AI 5 340 的 Framework 13 上、运行 Windows 11 时所得到的平均带宽。Linux 在这台笔记本上表现略差，但这让我感到意外，因为 Framework 自己针对我这台笔记本的端口文档说明它应当支持 USB 3.2 Gen 2x2——至少在端口 1 和端口 3 上是如此！

RTL8159 在许多 USB4 连接和所有 USB 3.2 Gen 2x1 连接上都会受到瓶颈限制。不幸的是，这会把带宽限制在远低于 8 Gbps 的水平。

我在我的 Framework 12 上进行了测试——它搭载较慢的 Intel 第 13 代移动 CPU——我发现它确实如文档所述支持 USB 3.2 Gen 2x2 速度，按理我应该能接近 10 Gbps。

只不过——至少在 Linux 上——并没有。该端口通过 lsusb 显示为 20000 Mbps（20 Gbps），但 iperf3 只让我达到 7 Gbps。我尝试下载并编译 Realtek 驱动，但它在 Ubuntu 26.04 上出错了，大概是因为该发行版的 Linux 内核（7.x）太新了。

于是我切换到 Windows 11，在用 USB Tree Viewer 确认端口显示为 Gen 2x2 之后，我得到的 iperf3 性能与在 Linux 上相同——至少在使用内置驱动时是这样。

不过在 Windows 上，Realtek 驱动安装没有任何问题，我终于得到了我一直想要的 9.4+ Gbps：

做双向测试时，我能得到约 9 Gbps 的上行和 4–5 Gbps 的下行，但在运行这些测试一段时间后，我遇到了一个新问题。模块变得非常烫。烫到我拿出了热成像相机来检查它：

底部塑料表面温度接近 70°C，虽然不会让你立刻被接触烫伤，但肯定会让你患上"烤皮肤综合征"（Toasted Skin Syndrome）——这让我想起过去 MacBook Pro 会在用户腿上留下印记的事情！

我就此询问了 WisdPi，他们表示塑料表面温度符合 IEC 62368-1 温度安全限值。只要你不让皮肤与该表面持续接触超过 10 秒，就没有问题。

但这是一台笔记本电脑。而我经常把它放在腿上使用！事实上，我现在就是坐在沙发上、用它在腿上写这篇博客文章……

当然，99% 的情况下当它在我腿上时，我用的是 WiFi。而且，该模块本身会从笔记本上凸出几厘米，所以如果你要用笔记本内胆包或贴合度较紧的包，就得把它取下来。

因此，在散热方面，我的建议是只在不会把它放在腿上使用的场景中使用这个模块。
