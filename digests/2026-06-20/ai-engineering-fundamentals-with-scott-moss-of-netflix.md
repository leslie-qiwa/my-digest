简介
章节时长：23 分钟
- Scott Moss 是 Netflix 的高级软件工程师，他在课程开篇将 AI 工程定义为一门在各公司中兴起的新学科。他首先预览了贯穿整个课程构建的项目，并讨论了如何通过评估、上下文工程和高级策略来改进该项目。
- Scott 讨论了 AI 工程师的实际工作内容，将其描述为广泛的、探索性的，与传统软件工程不同，因为 AI 系统通常是非确定性的。他还将 AI 工程定义为在基础模型之上构建应用程序，并在自然语言行为与更结构化的工程实践之间取得平衡。
- Scott 简要介绍了项目代码仓库。该应用程序是一个 AI 辅助的 Excalidraw 绘图应用。课程的重点将是构建和改进该 AI 智能体，使其能够根据用户提示生成图表。所有课程笔记也位于代码仓库中，笔记网站可以通过 `npm run docs` 运行。
Cloudflare 智能体
章节时长：1 小时 12 分钟
- Scott 详细讲解了在 Cloudflare 中运行的 AI 智能体的架构。Cloudflare 智能体是一个有状态的、长期存活的服务器端对象，构建在 Cloudflare 的 Durable Objects 之上。智能体循环将通过 WebSockets 接收来自用户的消息。当 LLM 处理消息时，响应将以流式传输回客户端。
- Scott 开始从零构建一个智能体。Zod 模式。他教学生如何使用 TypeScript 从零构建 AI 智能体，首先安装依赖并添加 OpenAI API 密钥。主要练习涉及在 schema.ts 文件中创建 TypeScript 类型，用于描述 Excalidraw 图表元素，如矩形、椭圆、箭头和文本。他解释说，他们将构建的两个工具——生成图表和修改图表——故意设计得简单粗糙，以便学生可以随着时间推移不断改进它们。他让学生手动编写第一个类型以熟悉结构，然后建议复制其余部分以节省时间。
- Scott 创建了用于添加和修改图表的工具。每个工具需要一个描述来告诉 LLM 何时使用它，一个用 Zod 定义的 inputSchema 来验证参数，以及一个当 LLM 调用该工具时执行的 execute 函数。这些工具适用于简单图表，但在复杂图表上会遇到困难。在课程后面，评分将准确显示智能体的不足之处。
- Scott 开始编写智能体代码。他利用 OpenAI ai-sdk 来实现智能体框架的大部分功能。一个静态系统提示被添加到智能体中，并建立了基类。
- Scott 实现了 onChatMessage 处理程序，该处理程序从 OpenAI SDK 返回 UI 消息流。`wrangler.toml` Cloudflare Worker 配置被更新以支持 Durable Object 绑定、迁移和 Node.js 兼容性。
- Scott 总结了 Cloudflare 智能体循环，并分析了 LLM 返回给客户端的 JSON。他还提供了关于 Cloudflare workers 和边缘运行时架构的更多细节。
聊天体验
章节时长：1 小时 9 分钟
- Scott 介绍了 useAgent 和 useAgentChat React hooks。Cloudflare Agents SDK 提供了这两个 hooks 来处理整个 WebSocket 连接和聊天协议。Scott 还概述了流式响应、模型如何选择工具，以及每次页面加载都会启动一个新的聊天会话。
- Scott 解释了画布如何与 AI 体验集成。当 `tool-generateDiagram` 或 `tool-modifyDiagram` 部分达到 output-available 状态时，其 output 字段包含我们需要通过 API ref 应用到 Excalidraw 画布的数据。整个过程将对每个工具输出应用一次，每当一个数据块到达时消息将重新渲染。
- Scott 构建了消息 UI。当从流式响应接收到各部分时，消息气泡将出现。当部分以 "tool-" 前缀开头时，将实例化一个 ToolStatus 组件，并使用工具的当前状态进行更新。
- Scott 将消息组件连接到聊天面板 UI。按钮的禁用状态也根据输入文本的存在或消息响应是否正在流式传输进行配置。
- Scott 引入了来自 Cloudflare Agents SDK 的 useAgentChat hook，并开始添加条件逻辑来管理响应的各个部分。如果部分类型等于 "tool-generateDiagram"，输出数组中的骨架元素将被转换为 Excalidraw 元素，以便添加到画布上。
- Scott 将聊天 UI 连接到应用程序。他通过生成新图表来测试工具的调用，然后要求智能体对其进行修改。修改受到限制，因为智能体无法"看到"画布上的内容。任何尝试向图表添加内容的操作都会生成一个全新的图表。
- Scott 花了几分钟回答与聊天 UI 功能相关的问题。他还更深入地探讨了 Zod 模式与智能体循环之间的关系。
评估与评分
章节时长：1 小时 54 分钟
- Scott 介绍了评估，并将其与软件工程中的传统单元测试进行比较。评估有助于建立基准线并在提示和模型变更时捕获回归问题。Scott 还讨论了建立黄金数据集的重要性，即精心策划的测试用例集合。
- Scott 介绍了在建立评估时需要理解的几个基本评分概念。这些包括手动与自动评分器、pass@k 和 pass^k，以及能力评估与回归评估。
- Scott 开始讨论评估框架。他介绍了测试用例、评估和评分结果的类型系统。他还探索了项目代码仓库中包含的黄金数据集。
- Scott 创建了评估框架，并提取了系统提示，以便在应用程序的 worker 端的智能体和 node 端的评估框架之间共享。评估框架加载黄金数据集，将每个测试用例直接通过模型运行（跳过 WebSocket 和 Durable Object 层），收集结果，并将其写入带时间戳的文件。
- Scott 完成了评估框架，实现了一个循环，遍历每个测试用例并将结果聚合到一个数组中。该数组被写入一个 JSON 文件。还添加了一些额外的性能计算，用于跟踪测试数量和平均测试时长。
- Scott 介绍了 Braintrust，一个用于在生产环境中衡量、评估和改进 AI 的 AI 可观测性平台。借助 Braintrust，工程师可以比较模型、迭代提示、捕获回归问题，并利用真实用户数据持续改进 AI 应用程序。
