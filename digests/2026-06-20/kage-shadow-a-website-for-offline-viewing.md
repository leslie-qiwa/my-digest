# Kage: Shadow a Website for Offline Viewing

一个网站，凝固为影

kage 在无头 Chrome 中渲染每个页面，抓取最终 DOM 的快照，移除所有脚本和事件处理器，并下载和重写 CSS、图片和字体。最终结果看起来与实际网站一模一样，但不执行任何代码：一个纯粹的 .html 文件夹，可以直接从磁盘打开。

用"另存为"保存页面，得到的副本仍会向外部发送请求，仍会运行分析代码，而且往往因为页面内容由 JavaScript 在运行时构建而显示为空白。kage（影）采用了相反的方式：它驱动一个真实的浏览器，按照人类实际看到的样子捕获页面，然后使其变为惰性。

假设你想在没有 Wi-Fi 的笔记本电脑上阅读 Paul Graham 的文章。一条命令即可镜像整个站点；另一条命令可以在离线状态下重新提供服务：

```
kage clone paulgraham.com
kage serve $HOME/data/kage/paulgraham.com
```

## 功能介绍

- **先渲染，再保存。** 每个页面都经过真实的无头 Chrome 渲染，因此由 JavaScript 组装内容的页面会被完整捕获，而非保存为空壳。
- **剥离所有脚本。** DOM 捕获完成后，kage 会移除所有 `<script>` 标签、每个 `on*` 事件处理器以及所有 `javascript:` URL。保存的页面不会发起任何网络请求，也不会执行任何代码。
- **保留布局。** 样式表、图片、字体和媒体文件会被下载并重写为相对本地路径，使离线副本看起来与原始页面一致。
- **可持续浏览。** 范围内的链接会被重写，指向其他已保存的页面，因此你可以像浏览实际网站一样在镜像中点击跳转。
- **打包为单一文件。** 可以将镜像压缩为单个 ZIM 归档（Kiwix 使用的开放格式），或一个自包含的可执行文件，运行后即可提供站点服务。

使用 `webview` 标签构建 kage，打包后的可执行文件会在独立窗口中打开，而非浏览器标签页，使离线镜像体验如同一个真正的应用程序。
