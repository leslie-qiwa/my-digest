**假设：** 政治家们失去了对舆论的控制，事后懊悔当初允许互联网自由开放。

**后果：** 政治家们需要说服科技公司_自愿_交还舆论控制权。

如何得出这个结论？很简单。要控制舆论，他们必须迫使你在网上所到之处都使用与财务信息绑定的真实身份。_好了……开始了。_

不久前，浏览器和插件还能轻松检测到多年前创建的用于识别成人内容和用户生成内容的标准。这对客户端开发者来说没有任何成本。[**yt-dlp 中的示例代码**](https://github.com/yt-dlp/yt-dlp/blob/master/yt_dlp/extractor/common.py#L1425 "yt-dlp 中查找 RTA 头部的示例代码：第1425行")。这对网站运营者来说也没有任何成本。这把管理孩子可以看到什么内容的责任放在了父母身上——这本来就是法律规定的父母职责所在。这一切在十多年前就已经存在，即便在当时实现起来也极其简单。它是什么样的呢？

**ICRA PICS 头部**

第一次尝试让网络对儿童更安全的方案是创建 PICS ICRA 标准。少数浏览器采用了它。一些第三方工具采用了它。一些网站服务器自愿实施了它。这次尝试可能对某些人来说过于复杂，因为它要求使用网页表单来生成一个详细描述网站所含内容类型的头部。由于操作摩擦，采用率停滞不前。我将省略杂乱的技术细节。

🔞 [**RTA 头部**](https://www.rtalabel.org/index.php?content=howtofaq#single "RTA 头部使用指南")

第二次尝试取得了更大的成功。为什么？因为摩擦——或者说，因为没有摩擦。任何人都可以在网页中或 HTTP 头部添加一个简单的头部标签，客户端——_浏览器、搜索引擎和其他爬虫_——就能立即知道该网站可能包含_不适合_儿童的内容（由设备所有者自行判断）。这可以是成人内容、用户生成内容或其他任何内容。无需具体说明。只是一个简单的机制，告诉客户端这是面向成年受众的。

HTML 指令如下所示：

`<meta name="rating" content="RTA-5042-1996-1400-1577-RTA">`

这可以嵌入任何网页或用于动态网页的页头和页脚中。本网页正在使用它。

NGinx 的 HTTP 头部如下所示：

`add_header Rating 'RTA-5042-1996-1400-1577-RTA' always;`

本站正在使用它。

在 HAProxy 中如下所示：

`http-response set-header Rating "RTA-5042-1996-1400-1577-RTA"`

简单。零成本。在服务器、负载均衡器和应用程序中设置起来极其简便。

所缺的只是需要将客户端代码重新加入浏览器，并可能同时添加到手机和平板电脑中。大多数开发者、公司和组织**今天**就能以极低的成本和工作量完成这项工作。对大多数人来说，这不过是一个简单的附带项目。
