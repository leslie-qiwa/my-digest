# The text in Claude Code’s “Extended Thinking” output

Claude Code 会将每次会话记录到磁盘。这些日志包含"思维块"——即模型工作时的自身推理过程。

这个周末我去检查了这些推理内容，结果发现只有一个长达 600 个字符的签名，没有任何文本。

于是我查阅了文档：https://platform.claude.com/docs/en/build-with-claude/extended-thinking

以下是一些值得了解的细节：
- Claude 将其推理过程加密为签名。
- Anthropic 持有密钥。你的机器无法获取。
- API 返回的是推理的**摘要**，而非推理本身。
- 获取完整的思维输出需要签署企业协议。

Matt Green 对此进行了深入研究，对签名块有更详细的观察。

在你向任何人承诺审计追踪之前，这些信息值得了解。另外请注意：通过 ctrl+o 获得的"扩展思维"输出是 Fable/Opus 思维的摘要，而非驱动模型在会话中实际行为的真实思维过程——这就像把 BMP 保存为 JPEG，然后编辑 JPEG 再存回 BMP，转换过程会造成数据丢失。[编辑：我最初把顺序写反了，引起了一些 HN 读者的反应。抱歉！]

Anthropic 对其应用行为的呈现方式令我失望。如果你需要记录**你的代理**在会话中使用的逻辑：
- 你无法通过本地文件还原推理逻辑。你系统上的推理日志对你来说是不可访问的。
- 你可以通过一些临时的抓取手段记录运行中 Claude Code 的输入、输出和操作——但即便如此，那也不是驱动代理行为的实际推理过程。

而且文档中的措辞非常含蓄。如果你还没喝咖啡，你可能会忽略"扩展思维返回的是 Claude 完整思维过程的摘要"这句话。

开源模型的性能提升需要更快到来。
